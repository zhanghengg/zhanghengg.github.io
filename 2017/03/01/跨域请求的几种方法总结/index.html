跨域请求的几种方法总结

2017-03-01 • 标签: Http

最近在使用ajax的时候有跨域请求的需求，所以查了一些资料，现在在这里做一下总结，大概有七种方法，不过只有前两三种是比较常用。

什么是跨域

讲到跨域请求的解决方法，首先要先了解什么是跨域，这要从浏览器的同源策略讲起，浏览器设计之初就规定了，出于安全性的考虑，不同源的请求不能互相访问，而不同源进行访问就是跨域。

简单的来说，同源指的是，协议，域名，端口三者相同时就是同源的，可以互相请求。三者只要有一者及以上不同，就是不同源的，默认情况下是不能够互相访问的。

如何跨域

虽然说浏览器为了安全性，不允许跨域访问，但是开发的时候有各种各样的需求，有时候就是需要跨域进行访问，这就有了各种解决跨域请求问题的各种方法。

1.CORS(跨域资源共享)

优点：1.支持get和post等多种请求方法.
      2. 使用简单，更为安全.
缺点：1.一些老版本的浏览器不支持，兼容性不够好.

CORS 是一个 W3C 标准，全称是”跨域资源共享”（Cross-origin resource sharing）它允许浏览器向跨源服务器，发出 XMLHttpRequest 请求。
CORS 需要浏览器和服务器同时支持才可以生效，对于开发者来说，CORS 通信与同源的 ajax 通信没有差别，代码完全一样。浏览器一旦发现 ajax 请求跨源，就会自动添加一些附加的头信息，有时还会多出一次附加的请求。

当发出跨域请求的时候，浏览器会在请求头添加一个Origin字段，该字段的值是发出这个请求的源（协议+域名+端口号），服务器根据Origin字段的值来判断是允许该请求访问。所以关键在于服务器的设置。

当服务器接收到其他源的http请求的时候，会拿到请求头的Origin字段的值跟自己响应头中的Access-Control-Allow-Origin字段进行比较，如果Access-Control-Allow-Origin中存在该源，那么服务器就会允许这个请求进行访问。所以要允许该源访问就应该在响应中设置Access-Control-Allow-Origin为该请求源的值，或者设置为*表示允许任意的源进行访问。

CORS还分为简单请求与非简单请求（大多数场景下我们使用的都是简单请求），以及在响应头中还有一些附加的设置字段，想要详细了解可以参考
跨域资源共享详解–阮一峰

2.JSONP(JSON with Padding)

优点：老版本的浏览器支持，兼容性比较好.
缺点：只支持get方法，使用没有CORS方便.

在html页面上加载js脚本不受同源策略的影响，所以我们可以利用这一特性，使用script标签实现跨域资源的请求。在页面中动态插入script，script标签的src属性就是后端api接口的地址，并且以get的方式将前端回调处理函数名称告诉后端，后端在响应请求时会将回调返还，并且将数据以参数的形式传递回来。

3.服务器代理

浏览器受到同源策略的限制，而服务器却不受限制，所以我们可以通过先向本地服务器发送请求，本地服务器再向不同源的服务器api发送请求，最后再把响应返回给浏览器，通过服务器中转代理的方式实现跨域请求。

4.postMessage

postMessage 是 HTML5 新增加的一项功能，跨文档消息传输(Cross Document Messaging)，目前：Chrome 2.0+、Internet Explorer 8.0+, Firefox 3.0+, Opera 9.6+, 和 Safari 4.0+ 都支持这项功能.该方法可以通过绑定window的message事件来监听发送跨文档消息传输内容。使用postMessage实现跨域的话原理就类似于jsonp，动态插入iframe标签，再从iframe里面拿回数据.

5.location.hash

在url中xxx.com#hash,#后面的就是hash值，改变hash值页面不会刷新，可以利用这一特性以及iframe来完成跨域传递数据。在一个域的页面创建一个隐藏的iframe，iframe的src指向另一个域的页面，此时src指向的页面就可以通过src中的hash值来获取传递的数据，然后在iframe指向的这个页面中再创建一个iframe，src指向第一个域的代理页面，通过这个页面来修改parent.loacation.hash的值来达到数据回传的目的。

6.window.name

window.name 不是一个普通的全局变量，而是当前窗口的名字，也就是说，不论页面怎么跳转，window.name都不会改变，所以可以通过在当前页设置window.name然后跳转到另一个页面获取window.name的值来达到跨域数据传递的目的。

7.document.domain

对于相同主域不同子域的页面来说，可以通过修改document.domain的方法来变成同源，从而符合浏览器的同源策略，达到数据传递的目的。
需要注意的是：

  1. 这两个域名必须属于同一个基础域名!而且所用的协议，端口都要一致，否则无法利用。
  2. document.domain只能赋成当前的域名或者基础域名。
  3. 如果页面修改了document.domain，则它包含的iframe，必须也设domain，才能进行交互。就算是同域的页面也必须要设。

总结

一般情况下，常用的跨域方法是CORS因为比较方便好用。在考虑低版本浏览器的兼容问题时候可以使用jsonp，但要注意jsonp只支持get方法。
