关于雅虎军规的学习

2017-01-20 • 标签: 性能优化

前提

随着互联网的发展，网页不再仅仅是十几年前的静态文字与图片的展示，用户对与网站的功能和性能的要求也越来越高，现在网站需要能够有丰富的与用户交互的功能和快速的访问速度，如何提高网站性能，成为炙手可热的问题。雅虎的35条军规提出了网站性能优化的标准，其中有14条核心标准。

1.尽可能减少HTTP的请求数

HTTP请求： 从客户端到服务端的请求消息,包括消息首行中对资源的请求方法、资源的标识符及使用的协议。
页面显示的所有文字、图片等资源以及js脚本和css样式表都是通过一个个http请求从服务器端获取，http请求数越多开销越大，减少http请求数可以提升网页的加载速度。
常用方法：

  1. 将多个css样式表合并
  2. 将多个javascript脚本文件合并
  3. 使用css sprites（图片精灵）将多张图片融合为一张大图，通过css的background-position属性来确定显示的图片
    ps：使用webpack，或者grunt可以实现自动合并

2.使用CDN（内容分发网络）: Use a Content Delivery Network

根据个人的理解，我认为内容分发网络就是在很多地方都放置了服务器，并且在每个服务器上都有你所上传到cdn上的文件的备份，当你访问这个资源的时候，系统根据你的网络的状况、拥挤程度等，自动选择一个访问速度最快的服务器负责提供资源。

3.添加Expire/Cache-Control头

越来越多的图片，脚本，css，flash被嵌入到页面中，当我们访问他们的时候势必会做许多次的http请求。其实我们可以通过设置Expires header 来缓存这些文件。Expire其实就是通过header报文来指定特定类型的文件在览器中的缓存时间。大多数的图片，flash在发布后都是不需要经常修改的，做了缓存以后这样浏览器以后就不需要再从服务器下载这些文件而是而直接从缓存中读取，这样再次访问页面的速度会大大加快。通过服务器端脚本可以完成Cache-Control和Expires的设置。

4.启用Gzip压缩：Gzip Components

把文件先在服务器端进行压缩，然后再传输。这样可以显著减少文件传输的大小。传输完毕后浏览器会 重新对压缩过的内容进行解压缩，并执行。目前的浏览器都能“良好”地支持 gzip。gzip的压缩比例非常大，一般压缩率为85%，就是 说服务器端100K的页面可以压缩到25K左右再发送到客户端

5.将css放在页面最上面

ie，firefox等浏览器在css全部传输完全之前不会去渲染任何的东西。IE，把样式表放在页面的底部的问题在于它禁止了网页内容的顺序显示。浏览器阻止显示以免重画页面元素，用户只能看到空白页了。Firefox不会阻止显示，但这意味着当样式表下载后，有些页面元素可能需要重画，这导致闪烁问题。所以我们应该尽快让css加载完毕。

6.将script放在页面最下面

  1. 浏览器解析是顺序执行的，为了防止script脚本的执行阻塞页面的下载，导致脚本以下的内容不被显示，为了避免这种影响用户体验的现象，应该把script脚本放在页面最下面，先将文档的内容渲染出来再执行脚本。
  2. 脚本阻塞并行下载数量，HTTP/1.1规范建议浏览器每个主机的并行下载数不超过2个,如果把图像文件分布到多台机器的话，可以达到超过2个的并行下载，但是当脚本文件下载时，浏览器不会启动其他的并行下载。

7.避免在CSS中使用Expressions

在css中使用Expressions，用户的操作经常会导致Expressions不必要的多次计算，严重影响网页的性能，造成重复的不必要的开销。

8.把javascript和css都放到外部文件中

  1. 这样使得代码易于维护。
  2. 虽然增加了http请求次数，但是减小了页面的大小，同时利用缓存，可以重复使用相应的文件。

9.减少DNS查询

DNS查询需要花费一定的时间,一般在20-120毫秒之间,浏览器不会下载该域名下的任何东西。所以减少dns查询的时间可以加快页面的加载速度，同时由于同一域名下的并行数量有限,折中考虑，一个页面所包括的域名最好在2-4个之间。

10.压缩 JavaScript 和 CSS

在发布版本的代码中去掉换行符，空格符，注释等不必要的字符可以减少页面的字节数，加载速度自然也就快了。而且压缩除了减少体积以外还可以起到一定的保护左右。

11.避免重定向

重定向是需要消耗时间的，每一次重定向就会增加一次web请求，所以因该尽量减少。

12.移除重复的脚本

重复的脚本不仅占用不必要的体积，而且在代码管理维护上也会带来麻烦。

13.使用ETag标签

ETag标签是一个实体标签，存在于http协议头部，使用特殊的字符串来标识某个请求资源的版本，当浏览器向服务器请求某个资源的时候，服务器会检查请求头部的ETag标签与本地资源的标签是否一致，如果一致则返回一个304状态码告诉浏览器使用本地的版本即可，如果不一致则将返回浏览器所请求的资源。并且利用ETag标签可以减少服务器不必要的重复计算，减轻服务器压力。

14.使用Ajax缓存

ajax的请求方法有get和post，post每次都会执行并且返回计算的结果，而get方法可以被缓存，我们可以根据需要来选择需要缓存的请求。
